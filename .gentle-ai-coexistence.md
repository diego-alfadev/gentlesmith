# Coexistence with gentle-ai

> How gentlesmith and gentle-ai write to the same files without stepping on each other.

## The golden rule

**Two namespaces, one file.** Each tool only writes inside its own marker namespace.

| Namespace | Owner | Example |
|---|---|---|
| `<!-- gentle-ai:NAME -->` | gentle-ai | persona, sdd, engram, etc. |
| `<!-- gentle-ai-overlay:NAME -->` | overlay tools (us, others) | `gentle-ai-overlay:gentlesmith` |

Anything outside both namespaces is user-authored and never touched.

## Who owns what

| Concept | gentle-ai | gentlesmith |
|---|---|---|
| Persona base (gentleman/neutral) | yes (skip with `--persona custom`) | replaces / overlays |
| Rules (commits, tools, safety, workflow) | partial | yes (overlay) |
| Env context (system, toolchain, deployment, secrets) | no | yes (overlay) |
| Engram MCP protocol | yes (always via sync) | no |
| SDD orchestrator (Agent Teams Lite) | yes (always via sync) | no |
| Skills bundled (sdd-init, sdd-apply, etc) | yes (always via sync) | no |
| MCP servers config | yes | no |
| Model routing / SDD profiles | yes | no |
| Custom skills personal to your workflow | no | yes (planned, out of `~/.claude/skills/`) |

We don't duplicate what gentle-ai already maintains upstream (engram, SDD, skills bundle). Gentlesmith is for the **personal layer**: persona, rules, env, sub-agent loadouts.

## Block format gentlesmith writes

```markdown
<!-- gentle-ai-overlay:gentlesmith -->

(content rendered by bin/distribute.ts)

<!-- /gentle-ai-overlay:gentlesmith -->
```

Single block per file. Idempotent — re-running `--apply` only rewrites this block.

## Pre-flight checklist when running `gentle-ai sync`

1. Did you set `--persona custom` in gentle-ai? Otherwise gentle-ai will inject its own persona above ours.
2. Run `gentlesmith` (dry-run) right after sync to verify the gentlesmith block survived. It should — gentle-ai's strip logic only targets its own legacy fingerprints.
3. If anything looks off, `gentlesmith --apply` re-renders cleanly. gentle-ai also keeps backups in `~/.gentle-ai/backups/{timestamp}/snapshot.tar.gz`.

## Why this boundary

gentle-ai already invests in maintaining engram, SDD, skills, and MCP plumbing aligned with their upstream. Duplicating those in gentlesmith means maintaining two copies — not worth it.

Persona, rules, and env are different: they're **personal**, don't evolve with upstream, and `--persona custom` (or just letting our overlay sit alongside the default persona) gives us a clean slot to fill.
