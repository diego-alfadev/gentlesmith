# Coexistencia con gentle-ai

> Cómo agents-system y gentle-ai conviven en los mismos archivos sin pisarse.

## La regla de oro

**`gentle-ai sync --persona custom` solo skipea la inyección de PERSONA.** Engram protocol, SDD orchestrator y skills bundled se siguen inyectando. Por eso agents-system **no tiene** fragmentos de engram ni SDD: serían duplicados.

## Quién posee qué

| Concepto | gentle-ai | agents-system |
|---|---|---|
| Persona / personality | NO (skipea con `--persona custom`) | **SÍ** — dentro de managed-block |
| Rules (commits, tools, safety, workflow) | NO | **SÍ** — dentro de managed-block |
| Env (system, toolchain, deployment, secrets) | NO | **SÍ** — dentro de managed-block |
| Custom skills personales | NO | **SÍ** — fuera del repo, en `~/.claude/skills/custom/` |
| Engram MCP protocol | **SÍ** (siempre, vía sync) | NO |
| SDD orchestrator (Agent Teams Lite) | **SÍ** (siempre, vía sync) | NO |
| Skills bundled (sdd-init, sdd-apply, etc) | **SÍ** (siempre, vía sync) | NO |
| MCP servers config | **SÍ** | NO |
| Model routing / SDD profiles | **SÍ** | NO |

## Managed-block format

Todo lo que agents-system escribe va envuelto en marcadores. Fuera del bloque, ni gentle-ai ni vos pierden contenido al re-renderizar.

```markdown
<!-- agents-system:start v1 -->

(contenido renderizado por distribute.ts)

<!-- agents-system:end -->
```

## Pre-flight checklist al hacer `gentle-ai sync`

1. ¿Tenés `--persona custom` activo en tu config de gentle-ai? Si no → activar.
2. ¿Querés que `sync` traiga skills bundled actualizadas, engram protocol actualizado, SDD orchestrator actualizado? Sí (eso es lo que querés que evolucione upstream).
3. Después del sync, ¿corriste `bun run distribute --dry-run` para verificar que tu managed-block sigue intacto? Recomendado las primeras veces.

## Si gentle-ai algún día rompe la convivencia

- **Backups automáticos**: gentle-ai tarballea todo a `~/.gentle-ai/backups/{timestamp}/snapshot.tar.gz` antes de cada sync. Rollback vía TUI de gentle-ai.
- **Worst case**: corrés `bun run distribute --apply` después del sync para re-aplicar tu managed-block.

## Por qué este boundary y no otro

Vos definiste como principio "no reinventar la rueda salvo justificación clara". Gentle-ai ya invierte en mantener su engram protocol y SDD orchestrator alineados con el upstream del proyecto — duplicarlos en agents-system significa mantener dos copias que vas a tener que sincronizar manualmente cada vez que gentle-ai actualice. No vale la pena.

Persona, rules y env son distintos: son **personales**, no evolucionan con upstream, y `--persona custom` ya nos da el hueco limpio para meterlos.
