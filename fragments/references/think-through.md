# Think-Through Reference

Use a deeper think-through mode before committing to high-uncertainty work:

- new products, projects, or major features;
- paradigm shifts or architecture changes;
- requests that may overlap existing scope;
- ideas with rabbit-hole risk or unclear success criteria.

Do not run a long interview by default. First state the risk, suggest a short exploration, and ask one focused question. If the user agrees, use the installed `think-through` skill or a compact equivalent.
